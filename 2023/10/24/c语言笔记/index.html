ertern int i和int i的区别

在C语言中extern int i; 是一个声明，它告诉编译器i是一个整数类型的变量，并且其定义在其他地方。这意味着i的内存分配并不发生在此声明处，而是在程序的其他地方。这通常用于当多个文件需要访问同一个全局变量时。

例如，假设你有两个文件file1.c和 file2.c。在 file1.c 中，你定义了一个全局变量int i = 0。然后在file2.c中，你想要访问这个在 file1.c 中定义的变量i，你就可以在 file2.c 中使用 `extern int i;来声明这个变量，这样，file2.c就可以访问和修改file1.c中定义的i了。

总的来说，extern关键字允许在一个文件中访问在另一个文件中定义的全局变量或函数。

注意，声明仅仅是告诉编译器这里有个 变量，但没有给变量分配内存空间。

关于 fgets和fputs函数

fgets的原型是char *fgets(char *str, int n, FILE *stream)

下面是fgets函数使用实例

fgets(x,6,fp)

x是一个字符串，fp是指向一个文本文件的指针，6是读取文件的数量（包括’\0’）。

fseek函数

有个fseek函数，我们可以将文件看做数组，在fopen打开的文件移动到任意字符处

fseek函数第一个参数是指向文件的指针，第二个是偏移量，表示从起始点开始要移动的距离，第三个参数是模式，模式用来表示起始点

有三个模式，以下已经列出

SEEK_SET 文件开始处

SEEK_CUR 当前位置

SEEK_END 文件末尾

下面是fseek函数的一些实例

fseek(fp,0,SEKK_SET) 这个表示了从文件开始处

fseek(fp,10L,SEKK_SET) 这个表示了从文件开始处左移第十个字节，不包含括号

fseek（fp,0,SEEK_END） 这个表示了从文件末尾处

关于fflush函数

fflush函数原型是 int fflush(FILE *fp)

目的是刷新缓冲区

如果长时间不刷新缓冲区，将会导致数据延迟输入和数据丢失，甚至数据混乱

其中，由于按位异或的特性：一个数两次按位异或以后等于原本的数，这样按位异或可以应用在密码当中。

按位与

如果你有两个数，A和B，那么A & B的结果是一个新的数，这个数的每一位都是A和B对应位上的结果。如果A和B的某一位都是1，那么结果的这一位就是1，否则就是0。跟逻辑与的操作很像

例如，假设我们有两个八位的二进制数A = 11001100和B = 10101010，那么A & B的结果就是10001000。你可以看到，只有在A和B的对应位都是1的时候，结果的这一位才是1。
